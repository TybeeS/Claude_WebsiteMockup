The feature I built is a search bar that helps users find particular flavored donuts in the menu. The search bar also has filters that get rid of the donuts within certain categories to make searches easier. 

The micro-iteration process felt easy to get into. Instead of making one single request for a feature and letting the AI go from there, it felt wiser to go through each step one by one and make sure the code generated was exactly what I wanted. It felt natural to get into.

Whenever I asked the AI to review its code, one issue I resolved was to add error handling/validation to catch missing attributes or DOM elements, as well as console warnings for debugging purposes.

Copilot Agent did its job well during the time I used it. I like how it can have direct access to GitHub repositories and make pull requests when told to. However, there were some instances where processes would not be completed when the AI said it would, and I had to manually command them to do the processes instead.



Many of the errors Copilot caught involved missing features or checks that would allow for better accessibility. One specific instance I caught was Copilot not styling the search feature in the same general color palette as my website.

Using AI on the terminal vs. the browser has its own strengths and weaknesses. The browser is more accessible and convenient to work with, but working with GitHub to make pull requests can be tedious to go through multiple times a session, when it would be easier to make iterations via the terminal.

Micro-iteration makes sense when you are generating particular features for a website. It is unnecessary when you are making an initial conceptual build that's more barebones.
